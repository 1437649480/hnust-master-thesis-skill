## 功能特性

- **全面格式检查** - 覆盖规范所有可自动检查项
- **智能格式修复** - 自动修复字体、字号、段落、引用标注、页眉页脚等格式问题，保留封面原始格式不变
- **页码分节管理** - 插入分节符，设置正确的页码格式（前置罗马数字 + 正文阿拉伯数字）
- **格式验证** - 逐项验证修复后的文档格式是否符合规范
- **示例文档生成** - 从零生成符合规范的示例docx文档
- **结构化报告** - 输出详细的检查报告，包含错误、警告和通过项

## 安全特性

**所有脚本均创建副本操作，不会修改原文件：**

| 脚本 | 输入文件 | 输出文件 |
|------|----------|----------|
| `check_thesis.py` | 论文.docx | 只读检查，无输出文件 |
| `fix_thesis.py` | 论文.docx | 论文_fixed.docx |
| `fix_pagenum.py` | 论文.docx | 论文_页码修复.docx |
| `generate_thesis.py` | 无 | 创建新文件 |
| `verify_thesis.py` | 论文.docx | 只读验证，无输出文件 |

## 规范覆盖

| 检查项 | 规范要求 |
|--------|----------|
| 页面设置 | A4纸张、上下30mm、左右25mm、页眉页脚20mm |
| 页眉 | "湖南科技大学硕士学位论文"，五号宋体居中 |

| 二级标题 | 4号宋体加粗(14pt)顶格，上下各空0.5行，单倍行距 |
| 三级标题 | 小4号宋体加粗(12pt)顶格，上下各空0.5行，单倍行距 |

## 快速开始

### 安装依赖

```bash
pip install python-docx
```

### 使用方法

#### 1. 检查格式
```bash
python scripts/check_thesis.py 论文.docx
```

#### 2. 修复格式
```bash
python scripts/fix_thesis.py 论文.docx [输出文件.docx]
```

#### 3. 修复页码（分节符）
```bash
python scripts/fix_pagenum.py 论文.docx
```

#### 4. 验证格式
```bash
python scripts/verify_thesis.py 论文.docx
```

#### 5. 生成示例文档
```bash
python scripts/generate_thesis.py 输出文件.docx
```

## 规范速查

### 页面设置
- 纸张：A4 (210mm×297mm)
- 页边距：上30mm、下30mm、左25mm、右25mm
- 页眉距离：20mm
- 页脚距离：20mm

### 页眉页脚
- 页眉：湖南科技大学硕士学位论文，五号宋体居中
- 页码：前置部分罗马数字，正文阿拉伯数字，格式"-X-"，5号TNR居中

### 标题格式
- 章标题：小2号宋体加粗(18pt)居中，上下各空一行，单倍行距
- 节标题：小3号宋体加粗(15pt)顶格，上下各空0.5行，单倍行距
- 二级标题：4号宋体加粗(14pt)顶格，上下各空0.5行，单倍行距
- 三级标题：小4号宋体加粗(12pt)顶格，上下各空0.5行，单倍行距

### 正文格式
- 小4号宋体(12pt)
- 段前段后不空行
- 1.25倍行距
- 首行缩进2字符(24pt)

### 关键词
- 数量：3-8个
- 关键词标签：小4号黑体
- 关键词词条：小4号宋体

### 参考文献
- 标题：小4号宋体加粗居中
- 条目：小5号宋体(9pt)，顶格，英文部分Times New Roman
- 标点：半角标点+空格
- 外文文献：不少于总文献数的1/3

### 图表标题
- 中英文双语
- 中文：宋体五号(10.5pt)
- 英文：Times New Roman五号(10.5pt)

### 另起页
- 中英文摘要、目录、每章、参考文献、附录、致谢需另起页

## 已知问题

| 问题 | 严重程度 | 说明 | 建议 |
|------|----------|------|------|
| 页码居中检测误报 | 低 | 某些文档中页码实际已居中但检测为未居中 | 手动检查页脚设置 |
| 公式居中检测不准确 | 低 | 对MathType或OMML格式的公式检测可能不准确 | 手动确认公式格式 |
| 引用标注误判 | 低 | 以（1）、（2）等开头的款/项标题可能被误判为引用标注 | fix_thesis.py已做排除处理 |
| 表格检测局限 | 低 | 复杂合并单元格表格检测可能不准确 | 手动检查表格格式 |
| 图表中英文双语检测 | 低 | 检测可能不准确 | 人工确认图表标题 |
| 页码前后缀格式 | 低 | verify_thesis.py检测页码"-X-"格式时可能有微小差异 | 以check_thesis.py结果为准 |

## 常见问题

### Q: 页码格式不对怎么办？
A: 先运行 `fix_pagenum.py` 插入分节符，再运行 `fix_thesis.py` 修复页脚字体。

### Q: 引用标注不是上标？
A: 运行 `fix_thesis.py` 自动将 [1], [2,3] 等设为上标。注意：以（1）、（2）等开头的款/项标题不会被误处理。

### Q: 字体显示不正确？
A: `fix_thesis.py` 同时设置中文字体和东亚字体，确保在Word中正确显示。

### Q: 文件被锁定？
A: 关闭Word后再运行脚本，或保存到不同文件名。

### Q: 修复后封面被修改了？
A: 请确保使用最新版本的 `fix_thesis.py`，已内置封面/扉页保护机制，修复过程不会修改封面内容。

### Q: 如何检查外文参考文献比例？
A: `check_thesis.py` 会自动检查外文参考文献是否不少于总文献数的1/3，并在检查报告中显示。
