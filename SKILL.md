# 湖南科技大学硕士学位论文格式 Skill

## 简介

此skill用于帮助检查和修复湖南科技大学硕士学位论文的格式，使其符合《湖南科技大学研究生学位论文撰写规范》。

**核心功能：**
- 全面检查docx文档格式是否符合规范
- 自动修复字体、字号、段落、引用标注、页眉页脚等格式问题
- 插入分节符并设置正确的页码格式（目录罗马数字 + 正文阿拉伯数字）
- 从零生成符合规范的示例docx文档
- 逐项验证修复后的文档格式

## 使用流程

典型使用顺序：

```
1. 检查格式 → python check_thesis.py 论文.docx
2. 修复格式 → python fix_thesis.py 论文.docx
3. 修复页码 → python fix_pagenum.py 论文.docx
4. 验证结果 → python verify_thesis.py 论文.docx
```

或从零生成规范文档：

```
python generate_thesis.py 硕士论文格式示例.docx
```

## 脚本说明

### 1. check_thesis.py — 格式检查

检查文档格式是否符合规范，输出结构化错误报告。

**覆盖的检查项：**
- 页面设置（A4、页边距、页眉页脚距离）
- 页眉（内容、字体、字号、居中）
- 页码（罗马/阿拉伯数字格式、字体、居中）
- 字体字号（章/节/条/款/正文各级标题和内容）
- 段落格式（行距、首行缩进、段前段后间距）
- 摘要格式（中英文标题、关键词标签、摘要内容字体、字数约600字）
- 关键词数量（3~8个）
- 引用标注（上标格式、不得在标题处）
- 公式格式（居中）
- 表格格式（表内文字字体字号）
- 插图格式（图序按章、图题在图下）
- 每章另起页（含摘要、目录、参考文献、致谢、附录）
- 标题规范（章标题≤25字、无标点符号）
- 参考文献（外文文献≥1/3）
- 内容结构完整性

**用法：** `python check_thesis.py 论文.docx`

### 2. fix_thesis.py — 格式修复

自动修复文档格式问题。修复过程保留封面/扉页原始格式不变。

**修复项目：**
- 页面设置（A4、页边距、页眉页脚距离）
- 页眉（五号宋体居中）
- 字体字号（按章/节/条/款/正文级别）
- 段落格式（行距、首行缩进、段前段后间距）
- 摘要格式（中英文标题、关键词标签）
- 致谢格式（标题小二号宋体加粗居中）
- 附录格式（标题小二号宋体加粗居中）
- 引用标注（设为上标格式）
- 参考文献格式（标题居中、条目顶格）
- 表内文字格式（中文五号宋体、英文五号TNR）

**用法：** `python fix_thesis.py 论文.docx [输出文件.docx]`

### 3. fix_pagenum.py — 页码修复

在文档中插入分节符，设置正确的页码格式。

**功能：**
- 检测文档是否有目录
- 在第一章前插入分节符
- 节1（目录等前置部分）：罗马数字页码 `-i-`, `-ii-`...
- 节2（正文部分）：阿拉伯数字页码 `-1-`, `-2-`...
- 页脚设置为 `-X-` 格式，五号Times New Roman居中

**用法：** `python fix_pagenum.py 论文.docx`

### 4. generate_thesis.py — 生成示例文档

从零生成符合规范的示例docx文档。

**生成内容：**
- 封面/扉页
- 中英文摘要+关键词
- 目录
- 正文：第一章引言 + 第二章演变历程 + 第三章实证分析 + 第四章国际比较 + 第五章结论
- 表格示例
- 参考文献15条
- 致谢
- 附录A：攻读学位期间发表的论文

**用法：** `python generate_thesis.py [输出路径]`

### 5. verify_thesis.py — 格式验证

全面验证修复后的文档格式，逐项对照规范检查。

**用法：** `python verify_thesis.py 论文.docx`

## 规范速查表

| 项目 | 要求 |
|------|------|
| 纸张 | A4 (210mm×297mm) |
| 上页边距 | 30mm |
| 下页边距 | 30mm |
| 左页边距 | 25mm |
| 右页边距 | 25mm |
| 页眉距离 | 20mm |
| 页脚距离 | 20mm |
| 页眉内容 | 湖南科技大学硕士学位论文 |
| 页眉字体 | 五号宋体(10.5pt)居中 |
| 页码格式 | 前置部分 -i- 罗马数字，正文 -1- 阿拉伯数字 |
| 页码字体 | 五号Times New Roman(10.5pt)居中 |
| 章标题 | 小2号宋体加粗(18pt)居中，上下各空一行(12pt)，单倍行距 |
| 节标题 | 小3号宋体加粗(15pt)顶格，上下各空0.5行(6pt) |
| 二级标题 | 4号宋体加粗(14pt)顶格，上下各空0.5行 |
| 三级标题 | 小4号宋体加粗(12pt)顶格，上下各空0.5行 |
| 条款项 | 小4号宋体加粗(12pt)，上下不空 |
| 正文 | 小4号宋体(12pt)，段前段后不空，1.25倍行距，首行缩进2字符 |
| 中文摘要标题 | 小2号宋体加粗居中 |
| 英文摘要标题 | 小2号Times New Roman加粗居中，另起页 |
| 关键词标签 | 中文：小4号黑体；英文：小4号Times New Roman加粗 |
| 关键词数量 | 3~8个 |
| 参考文献标题 | 小4号宋体加粗居中 |
| 参考文献条目 | 顶格，小5号宋体(9pt)，英文部分用Times New Roman |
| 外文文献 | 不少于总文献数的1/3 |
| 论文题目 | ≤25字，不设副标题 |
| 摘要字数 | 中文600字左右 |
| 引用标注 | 上标，小5号Times New Roman |
| 表格 | 三线表，表序按章(表X.Y)，表名居中无标点，中英文双语 |
| 插图 | 图序按章(图X.Y)，图题在图下，中英文双语 |
| 每章另起页 | 章标题前应有分页符；摘要、目录、参考文献、致谢、附录也需另起页 |

## 安全特性

**所有脚本均创建副本操作，不会修改原文件：**

| 脚本 | 输入文件 | 输出文件 |
|------|----------|----------|
| `check_thesis.py` | 论文.docx | 只读检查，无输出文件 |
| `fix_thesis.py` | 论文.docx | 论文_fixed.docx |
| `fix_pagenum.py` | 论文.docx | 论文_页码修复.docx |
| `generate_thesis.py` | 无 | 创建新文件 |
| `verify_thesis.py` | 论文.docx | 只读验证，无输出文件 |

## 常见问题

1. **页码格式不对**：先运行 `fix_pagenum.py` 插入分节符，再运行 `fix_thesis.py` 修复页脚字体
2. **引用标注不是上标**：运行 `fix_thesis.py` 自动将 [1],[2,3] 等设为上标
3. **字体显示不正确**：`fix_thesis.py` 同时设置中文字体和东亚字体，确保在Word中正确显示
4. **文件被锁定**：关闭Word后再运行脚本，或保存到不同文件名
5. **封面被修改**：确保使用最新版本的 fix_thesis.py，已内置封面保护机制
